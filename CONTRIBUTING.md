# Contributing

## Quick start
- Pick an issue labeled `good first issue` or `help wanted`
- Keep PRs small and focused
- Add tests for crypto changes

## What we value
- Threat model correctness
- Reproducible test vectors
- Clear docs and UX for emergencies

## PR checklist
- [ ] Added/updated tests
- [ ] No secret material committed
- [ ] README/spec updated if behavior changes
